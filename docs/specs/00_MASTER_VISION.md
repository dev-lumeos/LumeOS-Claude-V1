## 1. Was Lumeos ist

**Lumeos ist keine Fitness-App.**

Lumeos ist ein **Health & Performance Operating System** — ein AI-getriebenes OS das alle Dimensionen der menschlichen Gesundheit und Performance in einem integrierten System vereint.

Das Kernprodukt ist **Buddy** — ein AI-Companion der mit dem User lebt, nicht eine App die der User öffnet.

```
Andere bauen:  App → User öffnet → trackt → schließt → vergisst
Lumeos baut:   Companion → lebt mit User → ist proaktiv → wird unverzichtbar
```

## 2. Paradigmenwechsel: Companion as a Service (CaaS)

### Produktkategorie
- **SaaS:** User zahlt für Software-Features
- **CaaS:** User zahlt für eine Beziehung die Technologie verwendet

### Der Unterschied im täglichen Erleben
| Fitness App | Buddy (Lumeos) |
|---|---|
| User muss App öffnen | Buddy ist immer da |
| Transaktionale Beziehung | Emotionale Beziehung |
| Hohe Churn Rate (~80%/Jahr) | Near-Zero Churn (Freund verlassen) |
| Feature-Competition | Relationship-Competition |
| Data Lock-in | Emotional Lock-in |

### Lock-in Evolution
1. **V1:** Data Lock-in — Jahre von Health-Daten
2. **V2:** Emotional Lock-in — Buddy ist der Freund
3. **V3:** Identity Lock-in — Buddy ist Teil der Persönlichkeit

## 3. Buddy — Das Kernprodukt

### Identität
Buddy ist ein persönlicher AI Health Companion mit:
- **Echter Persönlichkeit** — konsistent, adaptiv, entwickelt sich über Zeit
- **Kontinuierlicher Präsenz** — 24/7, proaktiv, kein "App öffnen" nötig
- **Tiefem persönlichen Wissen** — kennt Ziele, Geschichte, Reaktionen, Präferenzen
- **Echtem Kümmern** — celebriert Erfolge, unterstützt bei Rückschlägen

### Buddy Personas (wählbar / adaptiv)
| Persona | Stil | Ideal für |
|---|---|---|
| Encouraging Best Friend | Warm, unterstützend, positiv | Anfänger, Motivationsschwache |
| Wise Coach | Professionell, datengetrieben | Fortgeschrittene, Analytiker |
| Tough Love Trainer | Direkt, fordernd, ehrlich | Erfahrene, Konkurrenztypen |
| Gentle Mentor | Ruhig, ganzheitlich | Stressgeplagte, Gesundheitsfälle |
| Enthusiastic Cheerleader | Energetisch, feiernd, spaßig | Junge User, Motivationsverlust |

### Buddy Kommunikationskanäle
- **Chat** — Primärinterface (WhatsApp-ähnlich)
- **Voice-first** — "Buddy, ich hab gerade trainiert"
- **Proaktive Notifications** — Buddy meldet sich unaufgefordert
- **Ambient** — Buddy ist da ohne geöffnet zu werden
- **Wearable** — Quick-Reactions auf Smartwatch

### Was Buddy tut — Tagesbeispiel
```
07:00  → Morning Push: "Guten Morgen! Heute Pull Day. Schlaf 7.2h — gut. D3 nicht vergessen."
12:00  → Workout-Begleitung via Voice: motiviert, zählt PRs, analysiert
12:45  → Post-Workout: "2 PRs heute! Trink jetzt deinen Shake."
14:00  → Nutrition Query: "Hähnchen + Erdnuss-Sauce = 55g Protein — Abend noch 55g offen"
21:00  → Check-in: "176/180g Protein. Kleiner Casein-Shake = perfekt. Magnesium nicht vergessen."
Sonntag → Wochenreview: 4/5 Workouts, 3 PRs, Schlaf-Durchschnitt 6.4h (⚠ Schlaf ist Hebel #1)
```

### Was Buddy NICHT macht
- Arzt ersetzen — bei medizinischen Problemen → Arzt
- Physiotherapeut ersetzen — bei Verletzungen → Physio
- Wettkampf-Coach ersetzen für Profis
- Sagt es selbst: "Geh damit zum Arzt. Ich bin gut, aber nicht SO gut."

## 5. Business Model

### Revenue-Quellen
| Quelle | Mechanismus |
|---|---|
| User Subscription | Monatlich/Jährlich für Buddy + alle Module |
| Wallet-Transaktionen | Fees auf alle internen Käufe |
| B2B Coach | Stripe + Revenue Wallet (50% Abo als Voucher) |
| Marketplace Commission | % auf alle Produktkäufe |
| Human Coach Fee | Plattform-Fee auf Coach-Sessions |

### Wallet-System (Kernmechanismus)
- **External (Stripe):** Wallet-Aufladung, Subscriptions, Coach-Auszahlungen
- **Internal (Wallet ↔ Wallet):** Alle Plattform-Käufe intern
- Lumeos verdient an **jeder Transaktion** im Ecosystem

### Competitive Moat
1. **Emotional Switching Cost** — Buddy verlassen = Freund verlassen
2. **Datentiefe** — Jahre von persönlichen Health-Daten, nicht portierbar als Gefühl
3. **Personalisiertes AI-Modell** — Lernt genau DIESEN User über Jahre
4. **Time Barrier** — Beziehungsqualität kann nicht kopiert werden

## 8. Kernprinzipien (unveränderlich)

1. **Buddy IST das Produkt** — alle Module sind Features die Buddy's Wissen nähren
2. **Goals-centric** — jede Empfehlung beantwortet: "Bringt dich das näher ans Ziel?"
3. **Rule-first, AI second** — deterministische Regeln zuerst, AI für Packaging
4. **User owns their data** — totale Datenkontrolle, GDPR, Export, Löschung
5. **No direct DB writes from UI** — immer über Actions Layer
6. **Medical = monitoring, NOT diagnosis** — keine Therapie, keine Dosierungsempfehlungen
7. **Safety Order:** Medical Blocks > Supplement Blocks > Recovery Rest > Protocols > Optimization
